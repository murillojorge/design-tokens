Naming Convention Reference

Token pattern: {category}.input.{element}.{variant}[.{size}][.{state}]
Categories declare the value type upfront: bg-color background fills  ·  color text, border, icon, and inline color  ·  text-size font size  ·  text-weight font weight
Default variant and default state are omitted from the name — bg-color.input.background.default is the resting fill. Error text is scoped to the error variant only: color.input.errorText.error. Type elements always carry the size suffix: text-size.input.fontSize.desktop / text-size.input.fontSize.mobile. Color elements are size-agnostic and vary by variant and state only.
References: Each token is an alias resolving to a primitive via {"{token.name}"} W3C DTCG notation. Color tokens reference color.{"{element}"}.{"{prominence}"}[.{"{state}"}] primitives. Type tokens reference type.size.{"{n}"} or type.weight.{"{n}"} primitives. Placeholder always resolves to neutral text regardless of variant to preserve legibility.
